## Overview

This project posts one AI-focused LinkedIn update per day using a rotating set of topics. Content is generated by Google Gemini (free) or Anthropic Claude (paid), deduplicated against your post history, and published via the LinkedIn REST API.

**What you get out of the box:**
- 5-topic content rotation — never repeats the same angle back-to-back
- Smart deduplication — SHA-256 hash + 60% Jaccard similarity check against recent posts
- Automatic OAuth token refresh — no manual re-auth for up to 365 days
- Rotating log files, SQLite post history, and dry-run mode for safe testing
- macOS launchd scheduling, cron, and GitHub Actions — pick what fits your setup

## How It Works

```
8:30 AM daily (launchd / cron / GitHub Actions)
        │
        ▼
 src/main.py  ─── get_next_topic()  ──►  Round-robin across 5 topics
        │
        ▼
 content_generator.py  ──►  Gemini / Claude API  ──►  Validate length & hashtags
        │
        ▼
 post_tracker.py  ──►  Dedup check (hash + Jaccard similarity)
        │
        ▼
 linkedin_client.py  ──►  POST /rest/posts  ──►  Retry on failure (3×, backoff)
        │
        ▼
 post_tracker.py  ──►  SQLite: record post + URN + status
        │
        ▼
 logs/autoposter.log  ──►  Rotating log (5 MB × 3 files)
```

**5 rotating topics:**

| Cycle Day | Topic |
|-----------|-------|
| 1, 6, 11… | AI Tips & Tricks |
| 2, 7, 12… | Claude Features & Updates |
| 3, 8, 13… | GitHub Copilot Tricks |
| 4, 9, 14… | Data Engineering + AI |
| 5, 10, 15… | Lessons Learned in AI |

## LinkedIn App Setup

This is the most important step. Do this before anything else.

### Step 1 — Create a Company Page (if you don't have one)

LinkedIn requires a Company Page to associate with a Developer App.

1. Go to [linkedin.com/company/setup/new](https://www.linkedin.com/company/setup/new)
2. Create a minimal page (e.g. "Your Name Projects") — takes 2 minutes

### Step 2 — Create a Developer App

1. Go to [linkedin.com/developers/apps](https://www.linkedin.com/developers/apps) → **Create app**
2. Fill in App name, associate your Company Page, upload a logo
3. Agree to terms → **Create app**

### Step 3 — Add Required Products

In your app → **Products** tab, request **both**:

| Product | Purpose |
|---------|---------|
| ✅ **Share on LinkedIn** | Grants `w_member_social` scope — **required for posting** |
| ✅ **Sign In with LinkedIn using OpenID Connect** | Grants `openid profile` scopes — used to fetch your Person URN during setup |

> "Share on LinkedIn" is typically approved instantly for personal use.

### Step 4 — Add Redirect URI

Go to **Auth** tab → **OAuth 2.0 settings** → add:
```
http://localhost:8080/callback
```

### Step 5 — Copy credentials to `.env`

Copy **Client ID** and **Client Secret** from the Auth tab into your `.env` file.

## Installation

```bash
# 1. Clone the repo
git clone https://github.com/DamyTheKnightKing/linkedin-autoposter.git
cd linkedin-autoposter

# 2. Copy and fill in credentials
cp .env.example .env
# Edit .env — add GEMINI_API_KEY, LINKEDIN_CLIENT_ID, LINKEDIN_CLIENT_SECRET

# 3. Run the full setup script
bash scripts/install.sh
```

The installer will:
- Create conda environment `linkedin-autoposter` (Python 3.11)
- Initialise the SQLite post history database
- Open your browser for LinkedIn OAuth (one-time)
- Run a dry-run to verify content generation works
- Install a macOS launchd job to post at 8:30 AM daily

## Finding Your LinkedIn Person URN

During OAuth setup, the app tries to auto-detect your Person URN. If it shows a warning, follow these steps:

### Option A — Automatic (recommended)

Run the OAuth setup after adding both LinkedIn products above:

```bash
/opt/miniconda3/envs/linkedin-autoposter/bin/python -m src.oauth_setup
```

With **both** products added, it will auto-detect and save your URN.

### Option B — From page source

1. Open your LinkedIn profile in a browser
2. Press `Cmd+U` (Mac) or `Ctrl+U` to view page source
3. Search (`Cmd+F`) for `urn:li:member:`
4. Note the alphanumeric value, e.g. `urn:li:member:edM1EcVerD`

Then set in `.env`:
```
LINKEDIN_PERSON_URN=urn:li:person:edM1EcVerD
```

> **Important:** The REST API requires the `urn:li:person:` prefix even though the page source shows `urn:li:member:`. Replace the prefix when setting the value.

### Option C — From the API error message

If your URN is unknown, send a test POST with your numeric ID in `urn:li:member:` format. The 422 error response will reveal the correct alphanumeric `urn:li:person:` URN:

```bash
TOKEN=$(grep LINKEDIN_ACCESS_TOKEN .env | cut -d= -f2)
curl -s -X POST "https://api.linkedin.com/rest/posts" \
  -H "Authorization: Bearer $TOKEN" \
  -H "LinkedIn-Version: 202601" \
  -H "Content-Type: application/json" \
  -d '{"author": "urn:li:member:YOUR_NUMERIC_ID", "commentary": "test", "visibility": "PUBLIC", "distribution": {"feedDistribution": "MAIN_FEED", "targetEntities": [], "thirdPartyDistributionChannels": []}, "lifecycleState": "PUBLISHED", "isReshareDisabledByAuthor": false}'
```

The error message will contain your correct URN.

## Scheduling Options

### Option A — macOS launchd (recommended)

Installed automatically by `install.sh`. Runs at 8:30 AM daily and catches up if your Mac was asleep.

```bash
# Check status
launchctl list | grep linkedin

# Unload (disable)
launchctl unload ~/Library/LaunchAgents/com.linkedin.autoposter.plist

# Reload (re-enable)
launchctl load ~/Library/LaunchAgents/com.linkedin.autoposter.plist
```

### Option B — Cron

```bash
crontab -e
# Paste contents of schedule/crontab.example
```

### Option C — GitHub Actions (cloud / no Mac required)

1. Push this repo to a **private** GitHub repository
2. Add all `.env` values as repository secrets (Settings → Secrets)
3. The workflow at `.github/workflows/daily_post.yml` runs at 8:30 AM SGT (UTC 00:30) daily

- Your persona and writing voice

## Token Lifecycle

| Token | Validity | Handling |
|-------|----------|----------|
| Access token | ~60 days | Auto-refreshed when < 24h remaining |
| Refresh token | ~365 days | Warning logged 30 days before expiry |

When the refresh token approaches expiry (⚠️ warning in logs), re-run:

```bash
/opt/miniconda3/envs/linkedin-autoposter/bin/python -m src.oauth_setup
```
